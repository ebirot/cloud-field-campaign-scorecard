Clean admin interface - Separate admin panel from main app

PROBLEM:
Admin section in main app displayed all UI elements (clouds sidebar,
quarters header) mixed with admin configs, creating cluttered UX.
User wanted clean admin interface without app elements visible.

SOLUTION:
- Replaced complex admin section with clean landing page
- Single CTA button opens full admin interface in new tab
- Complete separation: app UI stays in app, admin UI in dedicated page
- No sidebar/header pollution in admin section

CHANGES:

1. Main App (health_of_cloud_v2.html)
   - Simplified "Admin Panel" section (was "Admin Settings")
   - Removed inline configs (Cloud Visibility, Mappings grids, iframe)
   - Added clean landing page with:
     * Large lock icon (🔐)
     * Feature list (Analytics, Mappings, System, Cloud Config)
     * Single CTA button → opens /admin in new tab
   - No clouds sidebar visible in admin section
   - No quarters header visible in admin section

2. Admin Interface (admin_unified.html)
   - Dedicated admin sidebar (5 sections)
   - Complete separation from main app elements
   - Clean navigation: Overview, Users, Events, Mappings, System
   - "Back to app" button for easy return

USER FLOW:

Before:
App → Admin Mode → See clouds sidebar + quarters header + inline configs
= Cluttered, mixed UI

After:
App → Admin Mode → Clean landing page + 1 button
Button → New tab /admin → Dedicated admin interface
= Clear separation, focused UI

BENEFITS:
✅ Clean mental model: App = scorecards, Admin = management
✅ No UI pollution: Each interface optimized for its purpose
✅ Better UX: Users see only relevant elements
✅ Multi-tasking: Admin in separate tab while working in app
✅ Direct access: Can open /admin URL directly
